# 🚀 Social-to-Lead Agentic Workflow – AutoStream AI Agent

## 📌 Project Description
This project is a conversational AI agent built for a fictional SaaS product called **AutoStream**, which provides automated video editing tools for content creators.

The agent can understand user intent, answer product-related questions using a local knowledge base (RAG), and identify users who are interested in using the product. When a user shows high intent, the agent collects their details (name, email, and platform) and simulates lead capture using a backend function.

The goal of this project is to demonstrate how AI agents can convert user conversations into qualified business leads using intent detection, state management, and tool execution.

---

## 🔥 Features
- Intent Detection (Greeting / Query / High Intent)
- RAG-based Knowledge Retrieval
- Stateful Multi-turn Conversation
- Lead Capture Workflow
- Controlled Tool Execution

---

## 🧠 Architecture

This project follows an intent-driven conversational AI architecture.

When a user sends a message, the system first performs **intent detection** to classify it as a greeting, query, or high-intent request. Based on this classification, the agent decides the next action.

For general questions (like pricing or features), the system uses a **Retrieval-Augmented Generation (RAG)** approach. It fetches relevant information from a local knowledge base (JSON file) and returns accurate responses.

The system maintains conversation context using a custom **AgentState**, which helps track user progress across multiple steps. This is especially important for handling lead collection smoothly.

When a user shows strong interest (high intent), the agent switches to a **lead capture workflow**, where it collects user details step-by-step (name, email, platform).

Once all required information is collected, a backend function (**mock_lead_capture**) is triggered to simulate storing the lead.

This architecture ensures structured conversation flow, accurate responses, and controlled execution of actions.

---

## 📲 WhatsApp Integration

To integrate this AI agent with WhatsApp, we can use the WhatsApp Cloud API.

First, a webhook server is created to receive incoming messages from WhatsApp users. When a user sends a message, WhatsApp forwards it to our backend via the webhook.

The backend then passes the message to the AI agent, which processes it (intent detection, RAG, or lead flow) and generates a response.

To maintain conversation continuity, user state is stored using the user’s phone number as a unique identifier.

Finally, the response generated by the agent is sent back to the user through the WhatsApp API.

This setup enables real-time, two-way communication between users and the AI agent on WhatsApp.
